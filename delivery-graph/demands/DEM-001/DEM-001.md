# DEM-001: GED com workflow de aprovação para gestão de documentos de obras

Source: Demanda direta do usuário (Paulo), 2026-07-03

## Problem

Uma construtora precisa centralizar documentos (contratos, projetos, notas fiscais, licenças, laudos) espalhados por obra, com controle de quem acessa o quê, um fluxo formal de aprovação e trilha de auditoria. Hoje isso é feito de forma dispersa (arquivos soltos/e-mail) sem versionamento, controle de acesso por obra nem histórico confiável. Repositório greenfield: não há código a reutilizar (survey confirmou apenas README e .gitignore).

## Outcome

Sistema em produção real numa empresa; todo o baseline de segurança (bcrypt, JWT com refresh token, HTTPS, auditoria de toda ação, backup diário do banco e redundância de arquivos no MinIO) é obrigatório.

## Constraints

- Stack: Python
- FastAPI
- PostgreSQL
- MinIO (self-hosted via Docker)
- Docker Compose
- JWT
- structlog + health check. Ferramentas: ruff e pytest.
- Controle de acesso: relação usuário↔obra N:N + papel. Admin/Diretor veem todas as obras; Engenheiro/Financeiro só as atribuídas.
- A API é a superfície testada automaticamente (pytest); o Streamlit é fino e verificado manualmente.

## Non-goals

- Frontend Next.js/React polido — a UI do v1 é mínima em Streamlit; um frontend próprio fica como demanda futura.
- Storage genérico de arquivos — apenas PDF e imagens (PNG/JPG) de documentos de obra são aceitos.
- MFA obrigatório para todos — MFA é opt-in via TOTP no v1.
- Alta disponibilidade / clustering multi-nó — deploy é Docker Compose num host.
