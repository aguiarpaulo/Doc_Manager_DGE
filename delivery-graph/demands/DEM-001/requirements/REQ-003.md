# REQ-003

Demand: DEM-001
Priority: should

## Statement

O sistema deve oferecer MFA via TOTP que cada usuário pode ativar opcionalmente; quando ativo, o login exige o código TOTP além da senha.

## Acceptance

- Usuário ativa MFA e recebe segredo/QR TOTP
- Com MFA ativo
- login sem código válido é negado
- Com MFA ativo
- login com código TOTP válido é aceito

## Validation

Method: automated-test

Required evidence:

- Teste pytest cobrindo ativação
- login negado sem código e aceito com código (TOTP simulado)
