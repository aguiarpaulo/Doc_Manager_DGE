# REQ-001

Demand: DEM-001
Priority: must

## Statement

O sistema deve autenticar usuários por e-mail/senha, com senhas armazenadas via bcrypt, emitindo um JWT de acesso de curta duração e um refresh token para renovação.

## Acceptance

- Login com credenciais válidas retorna access token + refresh token
- Login com senha inválida retorna 401 e não vaza se o e-mail existe
- Senha é persistida como hash bcrypt
- nunca em texto puro
- Refresh token válido gera novo access token; refresh inválido/expirado retorna 401

## Validation

Method: automated-test

Required evidence:

- Teste pytest cobrindo login válido
- inválido
- hash bcrypt e ciclo de refresh (TestClient)
